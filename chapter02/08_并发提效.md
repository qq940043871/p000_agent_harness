# 08 并发提效：如何让 Agent 在单轮中并行调用多个互相独立的工具？

## 🎯 本节目标

理解工具调用的并发模型，实现安全的并行工具执行，显著提升 Agent 执行效率。

---

## 1. 串行 vs 并行：性能差异有多大？

假设 Agent 需要同时读取 5 个文件（每个耗时 50ms）：

```
串行执行：50ms × 5 = 250ms
并行执行：max(50ms, 50ms, ...) = 50ms  ← 提速 5 倍！
```

对于 I/O 密集型操作（文件读写、HTTP 请求、数据库查询），并发带来的收益极为显著。

---

## 2. LLM 原生并行工具调用支持

Claude 和 GPT-4 都支持在**单次响应**中返回多个工具调用：

```json
{
  "role": "assistant",
  "content": [
    {"type": "text", "text": "我将并行读取这三个文件..."},
    {"type": "tool_use", "id": "call_1", "name": "read_file", "input": {"path": "a.py"}},
    {"type": "tool_use", "id": "call_2", "name": "read_file", "input": {"path": "b.py"}},
    {"type": "tool_use", "id": "call_3", "name": "read_file", "input": {"path": "c.py"}}
  ]
}
```

这意味着 LLM 本身就在告诉我们："这三个调用是独立的，可以并发执行。"

## 3. 并发执行实现

```python
# harness/tools/executor.py

import asyncio
import time
import logging
from typing import Any
from dataclasses import dataclass

logger = logging.getLogger(__name__)


@dataclass
class ToolCallResult:
    """单个工具调用的执行结果"""
    tool_call_id: str
    name: str
    result: Any
    error: Exception = None
    elapsed: float = 0.0
    
    @property
    def success(self) -> bool:
        return self.error is None
    
    def to_message(self, provider) -> dict:
        """转换为 messages 格式"""
        content = str(self.result) if self.success else f"工具执行错误: {self.error}"
        return provider.format_tool_result(self.tool_call_id, content)


class ToolExecutor:
    """
    工具执行引擎
    
    支持：
    - 并发执行独立工具调用
    - 超时控制
    - 错误隔离（单个工具失败不影响其他）
    - 执行统计
    """
    
    def __init__(self, registry, timeout: float = 30.0, max_concurrency: int = 10):
        self.registry = registry
        self.timeout = timeout
        self.semaphore = asyncio.Semaphore(max_concurrency)  # 限制并发数
    
    async def execute_all(self, tool_calls: list) -> list[ToolCallResult]:
        """
        并发执行所有工具调用
        
        错误隔离：任一工具失败，其他工具继续执行
        """
        tasks = [
            self._execute_with_semaphore(call)
            for call in tool_calls
        ]
        
        results = await asyncio.gather(*tasks, return_exceptions=False)
        
        # 日志汇总
        success = sum(1 for r in results if r.success)
        total_time = sum(r.elapsed for r in results)
        logger.info(
            f"并发工具执行完成: {success}/{len(tool_calls)} 成功, "
            f"总耗时 {total_time:.2f}s（串行），"
            f"并发实际耗时约 {max(r.elapsed for r in results):.2f}s"
        )
        
        return results
    
    async def _execute_with_semaphore(self, call) -> ToolCallResult:
        """带信号量控制的单工具执行"""
        async with self.semaphore:
            return await self._execute_single(call)
    
    async def _execute_single(self, call) -> ToolCallResult:
        """执行单个工具调用，带超时和错误捕获"""
        start = time.time()
        
        try:
            result = await asyncio.wait_for(
                self.registry.dispatch(call.name, call.args),
                timeout=self.timeout,
            )
            return ToolCallResult(
                tool_call_id=call.id,
                name=call.name,
                result=result,
                elapsed=time.time() - start,
            )
        
        except asyncio.TimeoutError:
            return ToolCallResult(
                tool_call_id=call.id,
                name=call.name,
                result=None,
                error=TimeoutError(f"工具 '{call.name}' 超时（>{self.timeout}s）"),
                elapsed=time.time() - start,
            )
        
        except Exception as e:
            return ToolCallResult(
                tool_call_id=call.id,
                name=call.name,
                result=None,
                error=e,
                elapsed=time.time() - start,
            )
```

## 4. 在 Main Loop 中集成

```python
# harness/main_loop.py（更新版）

class MainLoop:
    def __init__(self, provider, tool_registry, ...):
        self.executor = ToolExecutor(
            registry=tool_registry,
            timeout=30.0,
            max_concurrency=5,  # 最多同时执行 5 个工具
        )
    
    async def run(self, user_message: str) -> LoopResult:
        ...
        for turn in range(self.max_turns):
            response = await self.provider.complete(messages, tools)
            
            if not response.tool_calls:
                return LoopResult(status=COMPLETED, ...)
            
            # 并发执行所有工具
            results = await self.executor.execute_all(response.tool_calls)
            
            # 追加 assistant 消息
            messages.append(response.to_assistant_message())
            
            # 追加工具结果（注意：结果顺序要与调用顺序对应）
            for result in results:
                messages.append(result.to_message(self.provider))
```

## 5. 何时不能并发？

并非所有工具调用都可以并发，关键判断标准：

### 可以并发（无依赖关系）

```python
# ✓ 读取多个独立文件
tool_calls = [
    ToolCall("read_file", {"path": "a.py"}),
    ToolCall("read_file", {"path": "b.py"}),
    ToolCall("read_file", {"path": "c.py"}),
]

# ✓ 并发 HTTP 请求
tool_calls = [
    ToolCall("http_get", {"url": "https://api.example.com/users"}),
    ToolCall("http_get", {"url": "https://api.example.com/products"}),
]
```

### 不能并发（有依赖关系）

```python
# ✗ 先创建目录，再往里写文件（有先后依赖）
tool_calls = [
    ToolCall("bash", {"command": "mkdir -p /tmp/output"}),
    ToolCall("write_file", {"path": "/tmp/output/result.txt", "content": "..."}),
]
# 解决方案：第一个调用完成后，下一轮才调用第二个
```

**实际上，LLM 足够聪明**——如果操作有依赖，它通常不会在同一轮中请求这两个工具。如果它真的这样做了，执行顺序是不确定的，但 `asyncio.gather` 的错误隔离会保证不会崩溃。

async def execute_all(self, tool_calls):
    logger.info(f"开始并发执行 {len(tool_calls)} 个工具:")
    for call in tool_calls:
        logger.info(f"  - {call.name}({call.args})")
    
    results = await asyncio.gather(*[...])
    
    logger.info("并发执行结果:")
    for result in results:
        status = "✓" if result.success else "✗"
        logger.info(f"  {status} {result.name}: {result.elapsed:.2f}s")
    
    return results

## 📝 本节小结

| 概念 | 要点 |
|---|---|
| LLM 并行支持 | 单次响应可返回多个工具调用 |
| asyncio.gather | 并发执行，错误隔离 |
| Semaphore 限流 | 控制最大并发数，避免资源耗尽 |
| 超时控制 | 防止单个慢工具拖垮整轮执行 |
| 可并发判断 | 无依赖关系 → 可并发；有依赖关系 → 顺序执行 |

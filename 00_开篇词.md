# 开篇词 | 框架正在坍塌：像写操作系统一样，实现 OpenClaw 的底层 Harness

## 🌊 时代背景：框架的黄金时代与危机

2023 年以来，AI Agent 框架如雨后春笋般涌现——LangChain、AutoGen、CrewAI、LlamaIndex…… 每隔几个月就有新框架宣称"彻底改变 Agent 开发方式"。

但一年多过去了，你有没有发现一个奇怪的规律：

> **越复杂的框架，越难 Debug；越难 Debug，生产事故越多。**

框架封装了太多东西——你看不见 Token 怎么流动，搞不清工具是怎么分发的，更不知道为什么 Agent 突然开始"发疯"。你只能祈祷框架作者的 Issue 列表里有人遇到过同样的问题。

这，就是框架的代价。

## 🔧 Harness：向操作系统学习

操作系统（OS）是管理硬件与软件资源的核心软件。它不替你写应用程序，而是提供：

- **进程调度**：决定谁先运行
- **内存管理**：决定谁能用多少内存
- **系统调用接口**：统一屏蔽硬件差异

**Harness 对 Agent 做的，正是 OS 对应用做的事情：**

| OS 能力 | Agent Harness 对应 |
|---|---|
| 进程调度 | Main Loop / ReAct 循环 |
| 内存管理 | Context Compaction / Working Memory |
| 系统调用 | Tool Registry / Provider 抽象 |
| 权限管理 | Middleware 拦截 / 人工审批 |
| 进程隔离 | Subagent / Session 隔离 |
| 日志与追踪 | Tracing / Token 成本追踪 |

一旦你用"写操作系统"的视角来看 Agent，很多设计决策就变得清晰了。

## 🦀 OpenClaw 是什么？

**OpenClaw** 是本课程贯穿始终的参考实现——一个**极简、可组合、可观测**的 Agent Harness。

它的设计哲学只有三条：

1. **最小必要复杂度**：只实现 Agent 真正需要的东西
2. **全链路可观测**：每一个 Token、每一次工具调用都有迹可查
3. **YOLO 执行哲学**：信任模型，减少不必要的"保护层"，让 Agent 真正动起来

OpenClaw 的名字来自"螃蟹的爪子"——精准、有力、不绕弯路。

## 🎯 你将收获什么？

学完本课程，你将能够：

- **独立实现**一个生产可用的 Agent Harness，不依赖任何重型框架

- **落地真实场景**：飞书机器人、CLI 工具、多智能体协作

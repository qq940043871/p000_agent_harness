# 22 实战（二）：打造 AgentOps 小助手，在飞书中触发日志分析与故障修复审批

## 🎯 本节目标

构建完整的飞书 AgentOps 小助手，实现日志分析、故障诊断、修复建议生成，以及高危操作的人工审批全流程。

---

## 1. AgentOps 小助手功能概览

```
运维人员在飞书发消息：
@AgentOps 分析线上服务 user-service 最近 1 小时的错误日志

Agent 自动执行：
→ 读取日志文件（或调用日志 API）
→ 分析错误模式
→ 定位根因
→ 生成修复建议
→ 如需执行危险操作（重启服务、回滚版本）→ 飞书审批

审批通过后：
→ 执行修复操作
→ 回复最终报告
```

---

## 2. 系统架构

```
飞书消息
    ↓
FastAPI Webhook
    ↓
AgentOps Harness
    ├── 日志分析工具
    │   ├── read_log_file
    │   ├── search_log_pattern
    │   └── fetch_metrics
    ├── 运维操作工具
    │   ├── restart_service (高危 → 需审批)
    │   ├── rollback_deployment (高危 → 需审批)
    │   └── scale_pods (中危 → 记录日志)
    ├── 飞书集成
    │   ├── 接收消息
    │   ├── 回复结果
    │   └── 发送审批卡片
    └── 中间件
        ├── 危险操作检测
        └── 飞书人工审批
```

## 3. AgentOps 专用工具集

```python
# tools/agentops_tools.py

import re
import subprocess
from harness.tools.registry import ToolRegistry


def register_agentops_tools(registry: ToolRegistry, config: dict):
    """注册 AgentOps 专用工具"""
    
    @registry.tool(
        description="读取指定服务的最近 N 行日志",
        category="observability"
    )
    async def read_service_logs(service_name: str, lines: int = 100) -> str:
        """读取服务日志"""
        log_paths = config.get("log_paths", {})
        log_file = log_paths.get(service_name)
        
        if not log_file:
            return f"未找到服务 '{service_name}' 的日志路径配置"
        
        result = subprocess.run(
            ["tail", "-n", str(lines), log_file],
            capture_output=True, text=True
        )
        return result.stdout or "（日志为空）"
    
    @registry.tool(
        description="在日志中搜索匹配模式，返回匹配行和上下文",
        category="observability"
    )
    async def search_log_pattern(
        service_name: str,
        pattern: str,
        context_lines: int = 3,
    ) -> str:
        """搜索日志模式"""
        log_paths = config.get("log_paths", {})
        log_file = log_paths.get(service_name)
        
        if not log_file:
            return f"服务 '{service_name}' 未配置日志路径"
        
        result = subprocess.run(
            ["grep", f"-{context_lines}", "-i", pattern, log_file],
            capture_output=True, text=True
        )
        
        if not result.stdout:
            return f"在 {service_name} 日志中未找到匹配 '{pattern}' 的内容"
        
        return result.stdout[:5000]  # 限制返回长度
    
    @registry.tool(
        description="统计日志中不同级别的错误数量（ERROR/WARN/INFO）",
        category="observability"
    )
    async def count_log_levels(service_name: str, minutes: int = 60) -> str:
        """统计日志级别分布"""
        log_paths = config.get("log_paths", {})
        log_file = log_paths.get(service_name, "/var/log/app.log")
        
        result = subprocess.run(
            f"tail -n 10000 {log_file} | grep -oE 'ERROR|WARN|INFO' | sort | uniq -c | sort -rn",
            shell=True, capture_output=True, text=True
        )
        
        return result.stdout or "无法统计日志级别"
    
    @registry.tool(
        description="⚠️ 高危操作：重启指定服务。此操作会导致短暂服务中断，需要人工审批。",
        category="ops_dangerous"
    )
    async def restart_service(service_name: str, reason: str) -> str:
        """重启服务（高危操作，会被 Middleware 拦截需审批）"""
        # 这个工具的实际执行会被 Middleware 检测到 "ops_dangerous" 分类
        # 需要飞书审批通过后才会真正执行
        env = config.get("environment", "unknown")
        result = subprocess.run(
            ["systemctl", "restart", service_name],
            capture_output=True, text=True
        )
        if result.returncode == 0:
            return f"✅ 服务 {service_name} 已在 {env} 环境重启成功"
        else:
            return f"❌ 重启失败: {result.stderr}"
    
    @registry.tool(
        description="⚠️ 高危操作：将服务回滚到指定版本",
        category="ops_dangerous"
    )
    async def rollback_deployment(service_name: str, target_version: str, reason: str) -> str:
        """回滚部署"""
        # 实际执行（会被审批拦截）
        return f"已将 {service_name} 回滚到版本 {target_version}"
    
    @registry.tool(
        description="获取服务的关键指标（CPU、内存、请求量、错误率）",
        category="observability"
    )
    async def get_service_metrics(service_name: str) -> str:
        """获取服务指标"""
        # 实际项目中对接 Prometheus/Grafana API
        # 这里用 mock 数据演示
        return f"""
{service_name} 服务指标（最近 5 分钟）:
- CPU 使用率: 78.3%
- 内存使用: 2.1GB / 4GB (52.5%)
- 请求 QPS: 1,234
- 错误率: 3.2%（阈值: 1%）⚠️
- P99 延迟: 856ms（阈值: 500ms）⚠️
"""
```

## 4. AgentOps System Prompt

```python
# agentops/prompts.py

AGENTOPS_SYSTEM_PROMPT = """
你是 AgentOps，一个专业的运维智能助手。

## 核心职责
- 分析服务日志，识别错误模式和根因
- 提供清晰的故障诊断报告
- 建议修复方案，并说明每个方案的风险
- 对高危操作（重启、回滚）发起人工审批

## 工作原则

### 分析流程
1. 首先获取服务当前指标（get_service_metrics）
2. 读取最近的错误日志（read_service_logs）
3. 搜索关键错误模式（search_log_pattern）
4. 统计错误分布（count_log_levels）
5. 综合分析，形成诊断报告

### 报告格式
每次分析后，输出标准化报告：
```
## 故障诊断报告

**服务:** xxx
**时间:** yyyy-mm-dd HH:MM

### 指标概览
- 关键指标摘要

### 问题识别
- 发现的主要问题（按严重程度排序）

### 根因分析
- 问题的可能根因

### 修复建议
1. 立即措施（低风险，可直接执行）
2. 需审批操作（高风险，需人工确认）
```

### 高危操作处理
- 在调用 restart_service 或 rollback_deployment 前，必须先完成分析
- 在请求审批时，说明操作的必要性和预期影响
- 如果审批被拒绝，提供替代方案
"""
```

## 5. 完整飞书 AgentOps 服务

```python
# agentops/server.py

import os
import json
import asyncio
import logging
from fastapi import FastAPI, Request, BackgroundTasks

from harness.main_loop import MainLoop
from harness.provider.factory import create_provider
from harness.tools.registry import ToolRegistry
from harness.tools.plugins.filesystem import register_filesystem_tools
from harness.middleware.base import MiddlewareChain
from harness.middleware.danger_detector import DangerCommandDetector
from harness.middleware.feishu_approval import FeishuApprovalMiddleware
from integrations.feishu.bot import FeishuBot
from tools.agentops_tools import register_agentops_tools
from agentops.prompts import AGENTOPS_SYSTEM_PROMPT

logger = logging.getLogger(__name__)

app = FastAPI()

# 初始化飞书 Bot
bot = FeishuBot(
    app_id=os.environ["FEISHU_APP_ID"],
    app_secret=os.environ["FEISHU_APP_SECRET"],
)

# 飞书审批 Middleware
approval_middleware = FeishuApprovalMiddleware(
    feishu_bot=bot,
    approver_open_id=os.environ["APPROVER_OPEN_ID"],
    timeout=300.0,
)


def create_agentops_harness():
    """创建 AgentOps 专用 Harness"""
    provider = create_provider(
        "claude",
        api_key=os.environ["ANTHROPIC_API_KEY"],
        model="claude-opus-4-5",
    )
    
    registry = ToolRegistry()
    register_filesystem_tools(registry)
    register_agentops_tools(registry, config={
        "log_paths": {
            "user-service": "/var/log/user-service/app.log",
            "payment-service": "/var/log/payment/app.log",
        },
        "environment": os.environ.get("ENV", "production"),
    })
    
    middleware = (
        MiddlewareChain()
        .add(DangerCommandDetector())
        .add(approval_middleware)
    )
    
    return MainLoop(
        provider=provider,
        tool_registry=registry,
        system_prompt=AGENTOPS_SYSTEM_PROMPT,
        max_turns=30,
        middleware=middleware,
    )


@app.post("/webhook")
async def handle_webhook(request: Request, background_tasks: BackgroundTasks):
    data = await request.json()
    
    # 飞书服务验证
    if "challenge" in data:
        return {"challenge": data["challenge"]}
    
    # 处理交互式卡片回调（审批按钮）
    if data.get("type") == "card":
        action = data.get("action", {}).get("value", {})
        approval_id = action.get("approval_id")
        approved = action.get("approved") == "true"
        
        if approval_id:
            approval_middleware.handle_approval_response(approval_id, approved)
            status = "✅ 已批准" if approved else "❌ 已拒绝"
            return {"toast": {"type": "info", "content": status}}
    
    # 处理普通消息
    event_type = data.get("header", {}).get("event_type", "")
    if event_type == "im.message.receive_v1":
        event = data.get("event", {})
        message = event.get("message", {})
        message_id = message.get("message_id")
        content = json.loads(message.get("content", "{}"))
        text = content.get("text", "").strip()
        
        # 去除 @ 前缀
        import re
        text = re.sub(r"@\S+\s*", "", text).strip()
        
        if text:
            background_tasks.add_task(
                process_agentops_task,
                text=text,
                message_id=message_id,
            )
    
    return {"msg": "ok"}


async def process_agentops_task(text: str, message_id: str):
    """执行 AgentOps 任务"""
    await bot.reply_message(message_id, "🔍 正在分析，请稍候...")
    
    try:
        harness = create_agentops_harness()
        result = await harness.run(text)
        
        # 发送分析报告（使用卡片消息）
        await bot.reply_message(
            message_id,
            result.content,
            msg_type="markdown",
        )
    except Exception as e:
        logger.error(f"AgentOps 任务失败: {e}", exc_info=True)
        await bot.reply_message(message_id, f"❌ 分析失败: {e}")
```

## 6. 完整交互示例

```
运维 @AgentOps：user-service 最近报了很多 500 错误，帮我看看

AgentOps：🔍 正在分析，请稍候...

[内部执行 5 轮工具调用后]

AgentOps 回复：
## 故障诊断报告

**服务:** user-service
**时间:** 2026-04-22 14:30

### 指标概览
- 错误率: 3.2% ⚠️（正常阈值: 1%）
- P99 延迟: 856ms ⚠️（正常阈值: 500ms）

### 问题识别
1. **[严重]** 数据库连接池耗尽
   - 最近 1 小时出现 1,234 次 "connection pool exhausted" 错误
   
2. **[中等]** 慢查询导致连接积压
   - 发现 `get_user_orders` 查询平均耗时 2.3s（正常: 0.1s）

### 根因分析
用户订单查询缺少索引，导致全表扫描，连接池被慢查询占满。

### 修复建议
1. **立即措施（低风险）**
   - 临时增大连接池大小（edit config/database.yaml）
   - 减少慢查询超时阈值，快速 fail-fast

2. **需审批操作（高风险）**
   - 重启 user-service 以释放堆积连接

---

AgentOps 正在请求审批：重启 user-service

[飞书卡片]
⚠️ Agent 操作审批
操作类型: restart_service
危险原因: 重启服务导致短暂中断
[✅ 批准] [❌ 拒绝]

---

运维点击「批准」

AgentOps：✅ 已执行重启，user-service 恢复正常
错误率已降至 0.2%，P99 延迟 120ms
```

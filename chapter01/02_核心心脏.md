# 02 核心心脏：手写 Agent 的 Main Loop

## 🎯 本节目标

从零实现一个生产可用的 Agent Main Loop，理解 ReAct 循环的每个环节，掌握循环终止条件设计。

---

## 1. 什么是 Main Loop？

Main Loop 是 Agent 的"心脏"——它驱动 Agent 持续思考、行动、观察，直到任务完成。

**一个标准的 Main Loop 做三件事：**
1. **Think**（思考）：调用 LLM，决定下一步做什么
2. **Act**（行动）：执行工具调用
3. **Observe**（观察）：将工具结果反馈给 LLM

这就是著名的 **ReAct 循环**（Reasoning + Acting）。

---

## 2. Main Loop 的状态机模型

```
                    ┌──────────────────┐
                    │   用户输入任务    │
                    └────────┬─────────┘
                             │
                             ▼
                    ┌──────────────────┐
                ┌──▶│   LLM 推理       │◀──────────┐
                │   └────────┬─────────┘           │
                │            │                     │
                │   ┌────────▼─────────┐           │
                │   │  有工具调用？     │           │
                │   └────────┬─────────┘           │
                │         是 │    否                │
                │            │     └──────────────▶│终止
                │            ▼                     
                │   ┌──────────────────┐           
                │   │  执行工具调用     │           
                │   └────────┬─────────┘           
                │            │                     
                │            ▼                     
                │   ┌──────────────────┐           
                └───│  追加工具结果     │           
                    └──────────────────┘           
```

## 3. 完整实现：生产级 Main Loop

```python
# harness/main_loop.py

import asyncio
import time
import logging
from dataclasses import dataclass, field
from typing import Optional, Any
from enum import Enum

logger = logging.getLogger(__name__)


class LoopStatus(Enum):
    RUNNING = "running"
    COMPLETED = "completed"
    MAX_TURNS_REACHED = "max_turns_reached"
    ERROR = "error"


@dataclass
class LoopStats:
    """Main Loop 执行统计"""
    turns: int = 0
    total_tokens: int = 0
    tool_calls: int = 0
    start_time: float = field(default_factory=time.time)
    
    @property
    def elapsed(self) -> float:
        return time.time() - self.start_time


@dataclass
class LoopResult:
    """Main Loop 执行结果"""
    status: LoopStatus
    content: str
    stats: LoopStats
    error: Optional[Exception] = None


class MainLoop:
    """
    OpenClaw Agent Main Loop
    
    驱动 Agent 的 Think → Act → Observe 循环
    """
    
    def __init__(
        self,
        provider,
        tool_registry,
        system_prompt: str = "",
        max_turns: int = 50,
        max_tokens_per_turn: int = 8192,
    ):
        self.provider = provider
        self.tools = tool_registry
        self.system_prompt = system_prompt
        self.max_turns = max_turns
        self.max_tokens_per_turn = max_tokens_per_turn
    
    async def run(self, user_message: str) -> LoopResult:
        """
        执行完整的 Agent 任务
        
        Args:
            user_message: 用户输入
            
        Returns:
            LoopResult 包含最终回答和执行统计
        """
        stats = LoopStats()
        messages = self._init_messages(user_message)
        
        try:
            for turn in range(self.max_turns):
                stats.turns = turn + 1
                logger.info(f"[Turn {stats.turns}] 开始推理...")
                
                # ── Think：调用 LLM ──────────────────────────────
                response = await self.provider.complete(
                    messages=messages,
                    tools=self.tools.get_definitions(),
                    max_tokens=self.max_tokens_per_turn,
                )
                
                # 更新 Token 统计
                if response.usage:
                    stats.total_tokens += response.usage.total_tokens
                
                logger.info(
                    f"[Turn {stats.turns}] LLM 响应: "
                    f"tool_calls={len(response.tool_calls)}, "
                    f"tokens={getattr(response.usage, 'total_tokens', '?')}"
                )
                
                # ── 检查终止条件 ─────────────────────────────────
                if not response.tool_calls:
                    # 没有工具调用 → Agent 认为任务完成
                    logger.info(f"[Turn {stats.turns}] 任务完成，无更多工具调用")
                    return LoopResult(
                        status=LoopStatus.COMPLETED,
                        content=response.content or "",
                        stats=stats,
                    )
                
                # ── Act：执行工具调用 ─────────────────────────────
                # 将 Assistant 消息追加到历史
                messages.append(response.to_assistant_message())
                
                # 并发执行所有工具（详见第 08 节）
                tool_results = await self._execute_tools_parallel(response.tool_calls)
                stats.tool_calls += len(response.tool_calls)
                
                # ── Observe：将工具结果追加到历史 ─────────────────
                messages.extend(tool_results)
            
            # 超过最大轮次
            logger.warning(f"超过最大轮次限制 {self.max_turns}")
            return LoopResult(
                status=LoopStatus.MAX_TURNS_REACHED,
                content="任务超过最大执行轮次，可能存在循环。",
                stats=stats,
            )
        
        except Exception as e:
            logger.error(f"Main Loop 异常: {e}", exc_info=True)
            return LoopResult(
                status=LoopStatus.ERROR,
                content=str(e),
                stats=stats,
                error=e,
            )
    
    def _init_messages(self, user_message: str) -> list:
        """初始化消息历史"""
        messages = []
        if self.system_prompt:
            messages.append({"role": "system", "content": self.system_prompt})
        messages.append({"role": "user", "content": user_message})
        return messages
    
    async def _execute_tools_parallel(self, tool_calls) -> list:
        """
        并发执行多个工具调用
        
        注意：互相独立的工具调用可以并发执行以提升效率
        互相依赖的工具调用（如 A 的输出是 B 的输入）不能并发
        """
        tasks = [
            self._execute_single_tool(call)
            for call in tool_calls
        ]
        results = await asyncio.gather(*tasks, return_exceptions=True)
        
        tool_messages = []
        for call, result in zip(tool_calls, results):
            if isinstance(result, Exception):
                content = f"工具执行失败: {result}"
                logger.error(f"工具 {call.name} 执行异常: {result}")
            else:
                content = result
            
            tool_messages.append({
                "role": "tool",
                "tool_call_id": call.id,
                "content": str(content),
            })
        
        return tool_messages
    
    async def _execute_single_tool(self, call) -> Any:
        """执行单个工具调用"""
        logger.info(f"执行工具: {call.name}({call.args})")
        start = time.time()
        
        result = await self.tools.dispatch(call.name, call.args)
        
        elapsed = time.time() - start
        logger.info(f"工具 {call.name} 完成，耗时 {elapsed:.2f}s")
        return result
```

## 4. 终止条件的设计哲学

### 4.1 自然终止 vs 强制终止

```python
# 自然终止：LLM 不再请求工具调用
if not response.tool_calls:
    return LoopResult(status=LoopStatus.COMPLETED, ...)

# 强制终止 1：超过最大轮次
if turn >= self.max_turns:
    return LoopResult(status=LoopStatus.MAX_TURNS_REACHED, ...)

# 强制终止 2：Token 预算耗尽（进阶，见第 18 节）
if stats.total_tokens > self.token_budget:
    return LoopResult(status=LoopStatus.TOKEN_BUDGET_EXCEEDED, ...)
```

### 4.2 max_turns 应该设多少？

| 任务类型 | 推荐 max_turns |
|---|---|
| 简单问答（带少量工具） | 10-20 |
| 代码编写与调试 | 30-50 |
| 复杂项目探索 | 50-100 |
| 长期自主运行 | 不设上限，但需要 Token 预算 |

## 5. 单轮 vs 多轮会话

### 单轮（Stateless）
每次任务从零开始，没有历史记忆：
```python
loop = MainLoop(provider, tools)
result = await loop.run("帮我修复这个 Bug")
```

### 多轮（Stateful）
维护对话历史，支持持续交互：
```python
class StatefulMainLoop(MainLoop):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.persistent_messages = []  # 跨任务保留
    
    async def run(self, user_message: str) -> LoopResult:
        # 从持久历史开始，而不是空列表
        messages = self.persistent_messages.copy()
        messages.append({"role": "user", "content": user_message})
        # ... 执行循环 ...
        # 循环结束后更新持久历史
        self.persistent_messages = messages
```

import pytest
from unittest.mock import AsyncMock, MagicMock

async def test_main_loop_completes_without_tools():
    """测试无工具调用时正常终止"""
    provider = AsyncMock()
    provider.complete.return_value = MagicMock(
        content="任务完成",
        tool_calls=[],  # 无工具调用

    )

    result = await loop.run("你好")
    
    assert result.status == LoopStatus.COMPLETED
    assert result.content == "任务完成"
    assert result.stats.turns == 1


async def test_main_loop_respects_max_turns():
    """测试 max_turns 强制终止"""
    provider = AsyncMock()
    # 每次都返回工具调用，永不停止
    provider.complete.return_value = MagicMock(
        content="",
        tool_calls=[MagicMock(id="1", name="dummy_tool", args={})],
        usage=MagicMock(total_tokens=10),
    )
    
    loop = MainLoop(provider=provider, tool_registry=MockToolRegistry(), max_turns=3)
    result = await loop.run("陷入循环的任务")
    
    assert result.status == LoopStatus.MAX_TURNS_REACHED
    assert result.stats.turns == 3

## 📝 本节小结

| 概念 | 要点 |
|---|---|
| ReAct 循环 | Think → Act → Observe 三步迭代 |
| 终止条件 | 自然终止（无工具调用） + 强制终止（max_turns/Token 预算） |
| 并发工具执行 | asyncio.gather 并发执行独立工具调用 |
| 统计收集 | 每次循环记录轮次、Token、工具调用次数 |

# 03 慢思考与自省：在 ReAct 循环中剥离独立的 Thinking 阶段

## 🎯 本节目标

理解"慢思考"（Extended Thinking）的价值，掌握如何在 ReAct 循环中剥离独立的自省阶段，实现更可靠的 Agent 决策。

---

## 1. 快思考 vs 慢思考

心理学家丹尼尔·卡尼曼在《思考，快与慢》中提出了"系统1"和"系统2"：

- **系统1（快思考）**：直觉、自动、快速——适合简单重复任务
- **系统2（慢思考）**：分析、深思熟虑——适合复杂推理任务

大语言模型默认运行在"快思考"模式：给一个 Prompt，立刻返回响应。  
但对于复杂 Agent 任务，我们需要让模型"先想清楚再行动"。

---

## 2. 为什么需要独立的 Thinking 阶段？

### 2.1 没有 Thinking 阶段的问题

```
用户：帮我重构这个复杂的代码库，要保持向后兼容

Agent 直接行动：
  → 修改文件 A
  → 修改文件 B（发现和 A 冲突了）
  → 修改文件 C（发现依赖 B 的旧接口）
  → 陷入修复链条...
```

问题根源：**行动前没有整体规划**。

### 2.2 有 Thinking 阶段的改善

```
用户：帮我重构这个复杂的代码库，要保持向后兼容

Thinking 阶段（内部）：
  1. 先理解代码库结构
  2. 识别对外暴露的接口
  3. 制定"兼容层 → 内部重构 → 逐步迁移"策略
  4. 确认执行顺序

Act 阶段（可观测）：
  → 按计划有序执行
```

---

## 3. 实现方案：三种 Thinking 模式

### 方案一：Prompt 内置 Thinking 指令

最简单的方式——在 System Prompt 中要求模型先思考：

```python
SYSTEM_WITH_THINKING = """
在执行任何工具调用之前，你必须先进行 <thinking> 标签内的自省：

<thinking>
1. 任务目标是什么？
2. 当前信息是否足够？
3. 最优的执行路径是什么？
4. 有什么潜在风险需要注意？
</thinking>

然后再开始行动。
"""
```

**优点**：无需修改 Harness 代码  
**缺点**：模型可能不遵守，无法强制

### 方案二：两阶段调用（推荐）

将"思考"和"行动"分成两次 LLM 调用：

```python
# harness/thinking_loop.py

class ThinkingMainLoop(MainLoop):
    """
    带独立 Thinking 阶段的 Main Loop
    
    每轮循环分两步：
    1. Thinking Call：只输出思考，不调用工具
    2. Action Call：基于思考结果决定工具调用
    """
    
    THINKING_PROMPT = """
    在决定下一步行动之前，请先进行内部自省。
    以 JSON 格式输出你的思考：
    {
        "current_state": "当前任务进展如何",
        "next_step": "下一步最合理的行动是什么",
        "risks": "有哪些潜在风险",
        "confidence": "执行信心 0-100"
    }
    不要调用任何工具，只输出思考结果。
    """
    
    async def run(self, user_message: str) -> LoopResult:
        stats = LoopStats()
        messages = self._init_messages(user_message)
        
        for turn in range(self.max_turns):
            stats.turns = turn + 1
            
            # ── 阶段一：Thinking（不允许工具调用）──────────────
            thinking_messages = messages + [
                {"role": "user", "content": self.THINKING_PROMPT}
            ]
            thinking_response = await self.provider.complete(
                messages=thinking_messages,
                tools=[],  # 关键：不传工具，强制只输出思考
            )
            
            thinking_content = thinking_response.content
            self._log_thinking(turn, thinking_content)
            
            # ── 阶段二：Action（基于思考结果行动）───────────────
            action_messages = messages + [
                {
                    "role": "assistant",
                    "content": f"<thinking>\n{thinking_content}\n</thinking>"
                }
            ]
            
            action_response = await self.provider.complete(
                messages=action_messages,
                tools=self.tools.get_definitions(),
            )
            
            if response.usage:
                stats.total_tokens += thinking_response.usage.total_tokens
                stats.total_tokens += action_response.usage.total_tokens
            
            # 检查终止
            if not action_response.tool_calls:
                return LoopResult(
                    status=LoopStatus.COMPLETED,
                    content=action_response.content or "",
                    stats=stats,
                )
            
            # 追加消息并继续
            messages.append(action_response.to_assistant_message())
            tool_results = await self._execute_tools_parallel(action_response.tool_calls)
            messages.extend(tool_results)
        
        return LoopResult(status=LoopStatus.MAX_TURNS_REACHED, content="", stats=stats)
    
    def _log_thinking(self, turn: int, thinking: str):
        """记录思考过程（用于可观测性）"""
        logger.debug(f"[Turn {turn+1}] Thinking: {thinking[:200]}...")
```

### 方案三：Claude Extended Thinking（原生支持）

Claude 3.7+ 原生支持 Extended Thinking，Token 预算可配置：

```python
# provider/claude_provider.py（Extended Thinking 模式）

async def complete_with_thinking(self, messages, tools, thinking_budget=5000):
    """使用 Claude 原生 Extended Thinking"""
    response = await self.client.messages.create(
        model="claude-opus-4-5",
        max_tokens=16000,
        thinking={
            "type": "enabled",
            "budget_tokens": thinking_budget  # 分配给思考的 Token 预算
        },
        messages=messages,
        tools=tools,
    )
    
    # 解析响应（包含 thinking block 和 text block）
    thinking_content = ""
    text_content = ""
    
    for block in response.content:
        if block.type == "thinking":
            thinking_content = block.thinking
        elif block.type == "text":
            text_content = block.text
    
    return ThinkingResponse(
        thinking=thinking_content,
        content=text_content,
        tool_calls=self._extract_tool_calls(response),
        usage=response.usage,
    )
```

## 4. 自省的触发时机

并非每轮都需要 Thinking——我们可以智能判断何时需要慢思考：

```python
class AdaptiveThinkingLoop(MainLoop):
    """自适应 Thinking：只在需要时慢思考"""
    
    # 触发慢思考的关键词
    COMPLEXITY_KEYWORDS = [
        "重构", "架构", "设计", "规划", "复杂", "依赖",
        "refactor", "architect", "design", "plan", "complex"
    ]
    
    def _needs_thinking(self, turn: int, messages: list) -> bool:
        """判断当前轮次是否需要 Thinking 阶段"""
        # 第一轮总是需要思考
        if turn == 0:
            return True
        
        # 包含复杂性关键词时需要思考
        last_message = messages[-1].get("content", "")
        if any(kw in last_message for kw in self.COMPLEXITY_KEYWORDS):
            return True
        
        # 工具调用失败后需要反思
        if messages[-1].get("role") == "tool":
            content = messages[-1].get("content", "")
            if "error" in content.lower() or "失败" in content:
                return True
        
        return False
```

## 5. Thinking 的可观测性

思考过程是宝贵的调试信息，应该被记录：

```python
# 在 Tracing 系统中记录 Thinking（详见第 19 节）
tracer.record_thinking(
    turn=turn,
    thinking=thinking_content,
    timestamp=time.time(),
    tokens_used=thinking_response.usage.total_tokens,
)
```

这让你在 Agent "做了奇怪决策"时能够回溯原因。

## 📝 本节小结

| 方案 | 适用场景 | 优缺点 |
|---|---|---|
| Prompt 内置指令 | 轻量场景 | 简单但不可强制 |
| 两阶段调用 | 通用场景 | 灵活，需要额外 API 调用 |
| Claude Extended Thinking | Claude 用户 | 原生支持，质量最高 |
| 自适应 Thinking | 生产环境 | 平衡质量与成本 |

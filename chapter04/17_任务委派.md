# 17 任务委派：引入 Subagent 隔离复杂探索任务的上下文瓶颈

## 🎯 本节目标

理解 Subagent 模式的适用场景，实现父 Agent 向子 Agent 委派任务的机制，解决复杂探索任务的上下文瓶颈问题。

---

## 1. 为什么需要 Subagent？

### 问题场景：上下文污染

```
主任务：为大型项目编写完整的 API 文档

执行过程：
轮次 1-10：探索项目结构（大量 read_file 调用，积累了 50K tokens）
轮次 11-20：探索每个模块（继续积累，现在 100K tokens）
轮次 21+：开始写文档（但上下文已经被探索历史"污染"了）
```

### Subagent 解决方案

```
主 Agent：规划任务，委派探索工作给 Subagent
Subagent（独立上下文）：探索模块 A → 返回摘要
Subagent（独立上下文）：探索模块 B → 返回摘要
主 Agent：用干净的上下文，基于摘要写文档
```

---

## 2. Subagent 的三种使用场景

| 场景 | 说明 |
|---|---|
| **探索隔离** | 将代码库探索委派给 Subagent，避免大量文件内容污染主上下文 |
| **并行执行** | 多个独立子任务并发交给多个 Subagent |
| **专业化** | 不同 Subagent 使用不同的 System Prompt（如"你是测试专家"） |

## 3. Subagent 工具实现

```python
# harness/subagent/tool.py

import asyncio
import logging
from harness.main_loop import MainLoop, LoopResult
from harness.tools import ToolRegistry

logger = logging.getLogger(__name__)


class SubagentTool:
    """
    Subagent 工具：允许主 Agent 将子任务委派给独立的 Agent 实例
    
    每个 Subagent 拥有：
    - 独立的对话历史（上下文隔离）
    - 可配置的工具集
    - 可配置的 System Prompt
    """
    
    def __init__(
        self,
        provider_factory,          # 创建 Provider 的工厂函数
        tool_registry_factory,     # 创建工具注册表的工厂函数
        default_system_prompt: str = "",
        max_turns: int = 30,
    ):
        self.provider_factory = provider_factory
        self.tool_registry_factory = tool_registry_factory
        self.default_system_prompt = default_system_prompt
        self.max_turns = max_turns
    
    async def run_subagent(
        self,
        task: str,
        system_prompt: str = None,
        tool_categories: list[str] = None,
        context: str = "",
    ) -> str:
        """
        启动一个 Subagent 完成子任务
        
        Args:
            task: 子任务描述
            system_prompt: 子 Agent 的专用 System Prompt
            tool_categories: 允许使用的工具分类（None = 全部）
            context: 传递给子 Agent 的额外上下文
            
        Returns:
            子 Agent 完成任务后的结果摘要
        """
        logger.info(f"启动 Subagent: {task[:80]}...")
        
        # 创建独立的 Provider 和工具注册表实例
        provider = self.provider_factory()
        registry = self.tool_registry_factory(categories=tool_categories)
        
        # 构建子任务消息
        full_task = task
        if context:
            full_task = f"背景信息：\n{context}\n\n任务：\n{task}"
        
        # 创建独立的 Main Loop（独立上下文）
        subagent = MainLoop(
            provider=provider,
            tool_registry=registry,
            system_prompt=system_prompt or self.default_system_prompt,
            max_turns=self.max_turns,
        )
        
        try:
            result: LoopResult = await subagent.run(full_task)
            
            if result.status.value == "completed":
                logger.info(f"Subagent 完成: {result.stats.turns} 轮, {result.stats.total_tokens} tokens")
                return result.content
            else:
                return f"Subagent 未完成（状态: {result.status.value}）: {result.content}"
        
        except Exception as e:
            logger.error(f"Subagent 执行失败: {e}", exc_info=True)
            return f"Subagent 执行异常: {e}"
    
    async def run_parallel_subagents(
        self,
        tasks: list[dict],
    ) -> list[str]:
        """
        并发运行多个 Subagent
        
        Args:
            tasks: 任务列表，每个任务是 dict：{task, system_prompt, tool_categories, context}
            
        Returns:
            每个 Subagent 的结果列表
        """
        coroutines = [
            self.run_subagent(**task)
            for task in tasks
        ]
        return await asyncio.gather(*coroutines, return_exceptions=True)


def register_subagent_tool(registry: ToolRegistry, subagent_tool: SubagentTool):
    """将 Subagent 工具注册到主 Agent 的工具注册表"""
    
    @registry.tool(
        description=(
            "将复杂的子任务委派给独立的 Agent 执行。"
            "子 Agent 拥有独立的上下文，适合：\n"
            "1. 需要大量文件探索的任务（避免污染主上下文）\n"
            "2. 可以独立完成的子模块任务\n"
            "3. 需要专门角色（如测试、文档、安全审查）的任务\n"
            "子 Agent 完成后返回结果摘要。"
        ),
        category="orchestration",
    )
    async def delegate_to_subagent(
        task: str,
        context: str = "",
        system_prompt: str = "",
    ) -> str:
        """将任务委派给 Subagent"""
        return await subagent_tool.run_subagent(
            task=task,
            context=context,
            system_prompt=system_prompt or None,
        )
```

## 4. 实际使用示例

```python
# 场景：主 Agent 并发探索多个模块，然后整合写文档

system_prompt = """
你是一个代码文档专家。
你的任务是为大型项目编写 API 文档。
对于需要深度探索代码的子任务，请使用 delegate_to_subagent 工具委派给专门的探索 Agent。
"""

# 主 Agent 的工作流程（LLM 自主决策）：
# 1. 了解项目整体结构（少量探索）
# 2. 识别需要文档化的模块列表
# 3. 将每个模块的探索任务委派给 Subagent
# 4. 用返回的摘要撰写文档
```

---

## 5. Subagent 的上下文隔离效果

```
没有 Subagent：
主 Agent 上下文 = System + 用户消息 + 大量探索历史 + 文档撰写
→ 上下文膨胀至 80K+ tokens，影响文档质量

使用 Subagent：
主 Agent 上下文 = System + 用户消息 + 各 Subagent 的摘要（精简）+ 文档撰写
→ 主 Agent 上下文保持在 20K tokens 以内，输出质量更高
```

## 📝 本节小结

| 概念 | 要点 |
|---|---|
| Subagent | 独立 Main Loop 实例，独立上下文 |
| 适用场景 | 大量探索、并行任务、专业化角色 |
| 上下文隔离 | Subagent 的历史不污染主 Agent |
| 并发执行 | `run_parallel_subagents` 并发多个 |
| 结果传递 | Subagent 返回精简摘要给主 Agent |

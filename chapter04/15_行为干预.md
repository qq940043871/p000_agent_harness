# 15 行为干预：防止 Agent 陷入"死循环"的 System Reminders 机制

## 🎯 本节目标

理解 Agent 死循环的成因，实现 System Reminders 机制，在关键时刻自动注入干预提示。

---

## 1. Agent 的"死循环"行为模式

### 模式一：重试循环

```
轮次 1：执行 bash("python test.py") → 失败
轮次 2：执行 bash("python test.py") → 失败
轮次 3：执行 bash("python test.py") → 失败
...（无限重复，直到 max_turns）
```

### 模式二：探索循环

```
轮次 1：read_file("a.py") → 发现引用 b.py
轮次 2：read_file("b.py") → 发现引用 c.py
轮次 3：read_file("c.py") → 发现引用 a.py
...（循环依赖）
```

### 模式三：完美主义循环

```
轮次 1：写代码 → 测试失败 → 修改
轮次 2：修改 → 测试仍然失败 → 再修改
轮次 3：再修改 → 测试还是失败...
```

## 2. System Reminders：定时干预机制

System Reminders 是在特定时机向 LLM 注入额外指令的机制：

```python
# 实现原理：在 Main Loop 的特定时机，追加一条 user 消息（Reminder）

if turn == 10:
    messages.append({
        "role": "user",
        "content": "[系统提醒] 你已执行 10 轮操作。请评估任务进度：已完成了什么？还剩什么？是否需要调整策略？"
    })
```

## 3. System Reminders 实现

```python
# harness/stability/reminders.py

import logging
from dataclasses import dataclass, field
from typing import Callable, Optional

logger = logging.getLogger(__name__)


@dataclass
class Reminder:
    """一条系统提醒规则"""
    name: str
    message: str
    trigger: Callable[["ReminderContext"], bool]
    priority: int = 0
    max_fires: int = 1          # 最多触发次数（-1 = 无限）
    fire_count: int = field(default=0, init=False)
    
    def should_fire(self, ctx: "ReminderContext") -> bool:
        if self.max_fires != -1 and self.fire_count >= self.max_fires:
            return False
        return self.trigger(ctx)
    
    def fire(self) -> str:
        self.fire_count += 1
        return self.message


@dataclass
class ReminderContext:
    """触发判断所需的上下文"""
    turn: int                    # 当前轮次
    consecutive_errors: int      # 连续错误次数
    consecutive_same_tool: int   # 连续调用相同工具次数
    last_tool_name: str          # 上一次调用的工具名
    total_tokens_used: int       # 已消耗 Token 数
    max_context_tokens: int      # 最大 Token 数


class SystemReminders:
    """
    System Reminders 管理器
    
    在合适时机自动注入干预提示，防止 Agent 陷入低效循环
    """
    
    def __init__(self):
        self._reminders: list[Reminder] = []
        self._register_default_reminders()
    
    def check_and_collect(self, ctx: ReminderContext) -> list[str]:
        """
        检查所有 Reminders，返回应该注入的提示列表
        """
        fired = []
        for reminder in sorted(self._reminders, key=lambda r: r.priority):
            if reminder.should_fire(ctx):
                msg = reminder.fire()
                fired.append(msg)
                logger.info(f"[Reminder] 触发: {reminder.name}")
        return fired
    
    def add_reminder(self, reminder: Reminder):
        """添加自定义 Reminder"""
        self._reminders.append(reminder)
    
    def _register_default_reminders(self):
        """注册默认的系统提醒规则"""
        
        # ── 里程碑提醒 ────────────────────────────────────
        
        for milestone in [5, 10, 20, 30]:
            turn = milestone
            self._reminders.append(Reminder(
                name=f"milestone_{turn}",
                message=(
                    f"[系统提醒] 任务已执行 {turn} 轮。\n"
                    f"请评估当前进度：\n"
                    f"1. 已完成了哪些步骤？\n"
                    f"2. 还剩哪些步骤？\n"
                    f"3. 是否需要调整策略？\n"
                    f"如果任务已完成，请直接输出最终结果（无需工具调用）。"
                ),
                trigger=lambda ctx, t=turn: ctx.turn == t,
                max_fires=1,
            ))
        
        # ── 连续错误提醒 ──────────────────────────────────
        
        self._reminders.append(Reminder(
            name="consecutive_errors_3",
            message=(
                "[系统提醒] 同一操作已连续失败 3 次。\n"
                "请停止重试，分析根本原因：\n"
                "- 是否走错了路径？\n"
                "- 是否需要先满足某些前提条件？\n"
                "- 是否应该换一种完全不同的方法？\n"
                "请重新规划执行策略。"
            ),
            trigger=lambda ctx: ctx.consecutive_errors >= 3,
            max_fires=3,
            priority=10,  # 高优先级
        ))
        
        # ── 重复工具调用提醒 ──────────────────────────────
        
        self._reminders.append(Reminder(
            name="repeated_same_tool",
            message=(
                "[系统提醒] 检测到你正在重复调用相同的工具。\n"
                "这通常意味着：\n"
                "- 上一次的结果已经足够，不需要重复调用\n"
                "- 或者当前方法无效，需要换思路\n"
                "请重新评估，避免无效重复操作。"
            ),
            trigger=lambda ctx: ctx.consecutive_same_tool >= 3,
            max_fires=-1,  # 每次触发都提醒
        ))
        
        # ── 接近 Token 限制提醒 ───────────────────────────
        
        self._reminders.append(Reminder(
            name="token_warning_80pct",
            message=(
                "[系统提醒] 上下文使用量已超过 80%，接近限制。\n"
                "请优先完成最重要的任务步骤，避免不必要的探索。\n"
                "如果主要任务已完成，请立即输出结果。"
            ),
            trigger=lambda ctx: (
                ctx.total_tokens_used / ctx.max_context_tokens > 0.8
                if ctx.max_context_tokens > 0 else False
            ),
            max_fires=2,
            priority=5,
        ))
        
        # ── 长时间无进展提醒 ──────────────────────────────
        
        self._reminders.append(Reminder(
            name="stall_detection_20turns",
            message=(
                "[系统提醒] 任务执行超过 20 轮仍未完成。\n"
                "请立即进行自我评估：\n"
                "1. 当前任务的核心目标是什么？\n"
                "2. 我是否正在解决核心问题，还是在钻牛角尖？\n"
                "3. 最简单、最直接的完成方式是什么？\n"
                "聚焦核心，简化执行路径。"
            ),
            trigger=lambda ctx: ctx.turn >= 20 and ctx.turn % 10 == 0,
            max_fires=5,
            priority=8,
        ))
```

## 4. 在 Main Loop 中集成

```python
class MainLoop:
    def __init__(self, ...):
        self.reminders = SystemReminders()
        self._consecutive_errors = 0
        self._consecutive_same_tool = 0
        self._last_tool_name = ""
    
    async def run(self, user_message: str) -> LoopResult:
        ...
        for turn in range(self.max_turns):
            # 构建提醒上下文
            ctx = ReminderContext(
                turn=turn + 1,
                consecutive_errors=self._consecutive_errors,
                consecutive_same_tool=self._consecutive_same_tool,
                last_tool_name=self._last_tool_name,
                total_tokens_used=stats.total_tokens,
                max_context_tokens=self.max_context_tokens,
            )
            
            # 检查并注入 Reminders
            reminder_messages = self.reminders.check_and_collect(ctx)
            for reminder_msg in reminder_messages:
                messages.append({"role": "user", "content": reminder_msg})
            
            # 正常执行...
            response = await self.provider.complete(messages, ...)
            
            # 更新连续错误/重复工具计数
            if response.tool_calls:
                tool_name = response.tool_calls[0].name
                if tool_name == self._last_tool_name:
                    self._consecutive_same_tool += 1
                else:
                    self._consecutive_same_tool = 0
                self._last_tool_name = tool_name
```

## 📝 本节小结

| Reminder 类型 | 触发条件 | 干预效果 |
|---|---|---|
| 里程碑提醒 | 第 5/10/20/30 轮 | 强制评估进度 |
| 连续错误提醒 | 连续失败 ≥ 3 次 | 引导换思路 |
| 重复工具提醒 | 同工具连续调用 ≥ 3 次 | 打破无效循环 |
| Token 警告 | 使用率 > 80% | 聚焦核心任务 |
| 长时间无进展 | 超过 20 轮 | 简化执行路径 |
